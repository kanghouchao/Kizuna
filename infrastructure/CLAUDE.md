# Infrastructure Conventions

- **Environment directories**: `infrastructure/development/` (HTTP only) and `infrastructure/release/` (HTTPS, Let's Encrypt, web→websecure redirect). Copy `infrastructure/.env.example` into each environment directory as `.env`. Switch with `task up env=release` (default is development).
- **`infrastructure/docker-compose.example.yml`** is the baseline compose template (same service structure as the per-environment files): apply structural service changes there first, then mirror them into `development/` and `release/`.
- **Traefik routing**: because `exposedByDefault: false`, a service to be exposed must carry the `traefik.enable=true` label (without it, it is silently not exposed). Paths are dispatched by `PathPrefix` (backend = `/api`, static = `/static`, frontend = everything else). If an app is unaware of its prefix, add a `stripPrefix` middleware (e.g. `backend-strip`).
- **DB / Redis**: referenced by the service names `database` / `cache`. Migrations are Liquibase (`backend/src/main/resources/db/changelog/`).
- **Env var naming**: the prefix names the product the setting *belongs to*, not the process that happens to read it — the backend is the only reader of `POSTGRES_HOST` and `MINIO_ENDPOINT`, but those settings describe PostgreSQL and MinIO, so they keep those prefixes. `APP_` is for settings that are the application's own policy (`APP_UPLOAD_MAX_FILE_SIZE`), not another product's. Prefixes in use: `POSTGRES_` / `MINIO_` / `REDIS_` / `HIKARI_` / `LETTUCE_` / `SPRING_` / `APP_`. One value gets exactly one name (the postgres image and the backend both read `POSTGRES_USER`; the MinIO container and the backend both read `MINIO_ROOT_USER`).
- **`backend` / `frontend` take no `env_file`**: they enumerate every variable they consume under `environment:`. Write `${VAR:-default}` only where the container topology makes the `application.yml` default definitively wrong (hostnames, ports, credentials, endpoints, `LIQUIBASE_CONTEXTS`) so a stale `.env` still boots (#311); list everything else as a bare name, which forwards the value when set and leaves the variable absent when not — letting the `application.yml` default apply instead of duplicating it here. Adding a new `${}` to `application.yml` therefore requires adding it to these `environment:` blocks too, or it will never reach the container. **A bare-name variable must never be declared empty in `.env`/`.env.example`**: absent means "variable not present, use the app default", but `VAR=` passes an empty string that silently overrides the default (Spring's `${VAR:default}` only falls back when the variable is undefined). Leave such lines commented out. Variables written as `${VAR:-default}` are immune, because `:-` treats empty as unset.
- **`release` requires its credentials explicitly**: `POSTGRES_USER` / `POSTGRES_PASSWORD` / `REDIS_PASSWORD` / `MINIO_ROOT_USER` / `MINIO_ROOT_PASSWORD` / `APP_JWT_SECRET` / `INITIAL_ADMIN_PASSWORD_HASH` use `${VAR:?message}`, so a missing value stops compose with a reason instead of silently starting production on a dev default (notably `INITIAL_ADMIN_PASSWORD_HASH`, whose fallback would seed the HQ admin with the well-known dev password). `development` keeps `${VAR:-default}` so a stale `.env` still boots (#311).
- **`LIQUIBASE_CONTEXTS` is environment-specific**: `development` defaults to `demo`, `release` to `production`. `.env.example` deliberately leaves it empty (it is copied into both environments) so the per-environment compose default wins.
- **MinIO console is `development`-only**, exposed at `/minio` (router `storage-console` → port 9001) and `development`'s `frontend` catch-all rule must keep excluding `/minio`. It needs **both** halves of the subpath setup: `MINIO_BROWSER_REDIRECT_URL` only injects `<base href="/minio/">` into the console HTML, which prefixes what the *browser* requests (assets and the console's own `/api/v1` calls alike, since its API base is derived from `document.baseURI`); MinIO's server-side routes stay at the root, so the `storage-console-strip` `stripPrefix=/minio` middleware is required to take the prefix back off. Drop either half and the console serves its SPA-fallback `index.html` for every asset, so the JS is parsed as HTML and the page renders blank except for the `noscript` text. That injected `<base href="/minio/">` is also the console's React Router basename, so a request for `/minio` **without** the trailing slash does not prefix-match it and the router renders nothing — a blank page that never even reaches `/login`. MinIO itself sends no redirect there (it answers `/minio` and `/minio/` with the same `index.html`), which is why the `storage-console-slash` `redirectRegex` middleware sends `/minio` → `/minio/` and must run *before* `storage-console-strip` in the chain. `release` deliberately has no such router: the console's only barrier is the MinIO root credential, so it must not sit on a public hostname. Publishing it remotely requires designing access control first.
- **Secrets**: `.env` must not be committed.
